# 后台管理模块设计

## 模块职责

提供Web管理后台，用于手动触发数据采集、查看系统状态、管理基础数据。

## 设计原则

- **简单直接**: 不做复杂的权限系统，单用户管理后台
- **手动触发**: 采集任务由管理员手动点击触发，不自动运行
- **状态可见**: 所有操作都有明确的状态反馈和日志

## API设计

### 管理操作接口

```
POST /api/v1/admin/import/stocks       触发股票导入
POST /api/v1/admin/import/concepts     触发概念导入
POST /api/v1/admin/import/concept-stocks 触发概念成分股关联
POST /api/v1/admin/collect/eastmoney   触发东方财富采集
POST /api/v1/admin/collect/cninfo      触发巨潮公告采集（Phase 2）

GET  /api/v1/admin/status              系统状态（DB大小、各表记录数）
GET  /api/v1/admin/logs                最近操作日志
```

### 管理数据查看接口（复用已有接口 + 新增）

```
GET  /api/v1/admin/stocks              股票列表（管理视图，分页）
GET  /api/v1/admin/news                资讯列表（管理视图，分页）
GET  /api/v1/admin/concepts            概念列表（管理视图，分页）
```

## 前端路由

```
/admin                 管理首页（Dashboard）
/admin/stocks          股票管理
/admin/news            资讯管理
/admin/concepts        概念管理
```

## 采集流程

手动触发 → 后端执行采集任务 → 返回执行结果 → 前端展示

```
用户点击[立即采集]
    ↓
POST /api/v1/admin/collect/eastmoney
    ↓
后端执行:
  1. 采集器抓取最新资讯
  2. 重要度过滤
  3. 实体识别 + 情感分析 + 事件检测
  4. AI摘要生成（如有API Key）
  5. 写入数据库
    ↓
返回结果: { collected: 50, filtered: 12, saved: 12 }
```

## 技术方案

- 前端: 复用Vue 3，新增admin路由和页面
- 后端: 新增admin API路由
- 操作日志: SQLite表 `admin_logs`
- 采集任务: FastAPI BackgroundTasks（异步执行，立即返回）

## 数据库新增

```sql
CREATE TABLE IF NOT EXISTS admin_logs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    action      TEXT NOT NULL,           -- 操作类型
    detail      TEXT,                    -- 操作详情
    status      TEXT DEFAULT 'success',  -- success/failed/running
    created_at  TEXT DEFAULT (datetime('now'))
);
```
